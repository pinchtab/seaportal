## Frequently Asked Questions

How does SeaPortal handle JavaScript-rendered content?

SeaPortal uses a headless browser engine to fully execute JavaScript, waiting for dynamic content to load before extraction. It detects network idle and DOM stability to ensure all async content is captured.

Can I extract content from pages behind authentication?

Yes. SeaPortal supports cookie injection and session management, allowing you to extract content from authenticated pages. Enterprise plans include built-in credential vaulting.

What makes the accessibility tree approach unique?

Unlike raw HTML parsing, our accessibility tree extraction captures the semantic meaning of content as perceived by assistive technologies. This provides cleaner, more structured output ideal for AI processing.

## Developer Resources

Everything you need to integrate SeaPortal into your applications. 4 official SDKs, 3 CLI tools, and 3 API endpoints at your disposal.

### Official SDKs

#### Python — seaportal-py

**Version:** 2.4.1

The official Python SDK provides a Pythonic interface to SeaPortal extraction. Features async/await support, pandas DataFrame integration, and automatic retry handling with exponential backoff.

```
from seaportal import Client; client = Client(); result = await client.extract("https://example.com")
```

#### TypeScript — @seaportal/sdk

**Version:** 3.1.0

Full TypeScript support with complete type definitions for all extraction options and response structures. Works seamlessly with Node.js, Deno, and modern bundlers.

```
import { SeaPortal } from "@seaportal/sdk"; const sp = new SeaPortal({ apiKey }); const doc = await sp.extract(url);
```

#### Go — github.com/seaportal/go-client

**Version:** 1.8.0

Lightweight Go client optimized for high-throughput extraction pipelines. Supports connection pooling, context cancellation, and streaming responses for large documents.

```
client := seaportal.NewClient(apiKey); doc, err := client.Extract(ctx, url, seaportal.WithAccessibilityTree())
```

#### Rust — seaportal-rs

**Version:** 0.9.2

Community-maintained Rust crate offering zero-copy deserialization and compile-time extraction schema validation. Ideal for performance-critical data pipelines.

```
let client = SeaPortal::new(api_key)?; let doc = client.extract(&url).await?;
```

### CLI Tools

#### sp-extract

Command-line extraction tool for quick one-off jobs and shell scripting. Supports output to stdout, files, or piping directly to other tools like jq.

```
sp-extract https://example.com --format json --selector main
```

#### sp-watch

Monitor URLs for content changes with configurable polling intervals. Emits structured diffs and can trigger webhooks or local scripts on detected changes.

```
sp-watch --url https://example.com/pricing --interval 1h --on-change ./notify.sh
```

#### sp-batch

Process URL lists in parallel with automatic rate limiting and failure recovery. Ideal for large-scale extraction jobs with progress reporting.

```
sp-batch --input urls.txt --output results/ --parallelism 10
```

### API Reference

#### POST /v2/extract

Primary extraction endpoint. Accepts a URL and configuration options, returns structured content in the specified format. Supports both synchronous and webhook-based async modes.

#### GET /v2/jobs/{id}

Poll job status for async extractions. Returns progress percentage, estimated completion time, and partial results when available.

#### POST /v2/batch

Submit multiple URLs for batch processing. Returns a batch ID for tracking overall progress and retrieving individual results as they complete.
