The SeaPortal stealth mode uses a multi-layer approach to browser fingerprint normalization. Each layer addresses a specific detection vector commonly used by anti-bot systems.

Layer 1: Navigator Properties

Override navigator.webdriver, navigator.plugins, and navigator.languages to match typical human browser configurations. The webdriver property is the most commonly checked indicator of automation.

Layer 2: Chrome Runtime

Inject a mock chrome.runtime object that passes existence checks without exposing automation-specific APIs. Many sites check for chrome.runtime.connect availability.

Layer 3: Timing Analysis

Introduce micro-delays in event handling to simulate human reaction times. Bots typically respond to events with sub-millisecond precision, while humans exhibit natural variance in the 50-300ms range.

Layer 4: Canvas Fingerprint

Apply subtle noise to canvas rendering operations to prevent fingerprint matching against known headless browser signatures. The noise is imperceptible but unique per session.
